Chrome DevTools MCP Server

强大的浏览器开发者工具服务器，提供高级网页调试和自动化能力

什么是 DevTools MCP Server？

DevTools MCP (Message Control Protocol) Server 是一个基于 Chrome DevTools Protocol 的高级服务器实现，它提供了一套完整的 API，允许开发者通过程序化方式控制和调试浏览器。这使得开发者可以自动化测试、执行复杂的调试任务，以及创建自定义的开发工具和工作流。

🔧

程序化控制

通过 API 完全控制浏览器的各种功能，包括 DOM 操作、CSS 调试、JavaScript 执行等。

🤖

自动化测试

实现端到端测试、UI 自动化和性能测试，确保网页应用质量和性能。

⚡

开发工作流

创建自定义开发工具和工作流，提高开发效率和代码质量。

核心功能

01

DOM 操作

通过 API 动态查询、创建、修改和删除 DOM 元素，实现复杂的页面交互自动化。

02

CSS 调试

实时修改和调试 CSS 样式，查看样式应用效果，解决布局和样式问题。

03

JavaScript 执行

在页面上下文中执行 JavaScript 代码，获取执行结果，调试和分析运行时行为。

04

网络监控

监控和分析网络请求，模拟不同网络条件，优化页面加载性能。

05

性能分析

捕获性能指标，分析渲染性能，识别和解决性能瓶颈。

06

自动化测试

编写和执行端到端测试脚本，验证应用功能和用户体验。

主要工具和API

页面操作工具

  • navigate_page: 导航到指定 URL，支持前进、后退和刷新操作
  • take_screenshot: 捕获页面或特定元素的截图
  • take_snapshot: 获取页面 DOM 结构快照
  • resize_page: 调整页面视口大小，用于响应式设计测试

DOM 交互工具

  • click: 模拟鼠标点击操作
  • fill: 在输入框中填写文本
  • fill_form: 批量填写表单字段
  • drag: 模拟拖拽操作
  • hover: 模拟鼠标悬停效果

调试和分析工具

  • evaluate_script: 在页面上下文中执行 JavaScript
  • list_console_messages: 获取控制台日志信息
  • list_network_requests: 查看网络请求详情
  • performance_start_trace: 开始性能分析追踪
  • performance_stop_trace: 停止并获取性能分析结果

使用场景示例

自动化功能测试

使用 DevTools MCP 可以编写自动化脚本来测试用户界面的功能，模拟用户交互并验证结果。

// 示例：登录功能测试
evaluate_script({
  function: () => {
    document.querySelector('#username').value = 'testuser';
    document.querySelector('#password').value = 'password';
    document.querySelector('#login-form').submit();
    return document.URL; // 返回登录后的URL
  }
});

性能监控

通过性能追踪API，可以监控网页的加载性能，收集关键性能指标，优化用户体验。

// 示例：性能监控
performance_start_trace({ reload: true, autoStop: true });
// 等待页面加载完成
// ...
performance_stop_trace();
// 分析性能报告

响应式设计测试

调整浏览器视口大小，测试网页在不同设备上的显示效果，确保响应式设计正确实现。

// 示例：测试不同设备尺寸
resize_page({ width: 375, height: 812 }); // iPhone X
// 截图保存
take_screenshot({ filePath: 'mobile-view.png' });
resize_page({ width: 768, height: 1024 }); // iPad
take_screenshot({ filePath: 'tablet-view.png' });

开发最佳实践

📝

编写模块化测试

将测试代码拆分为可复用的模块，便于维护和扩展，提高测试代码的可读性。

⏱️

使用适当的等待策略，避免硬编码延迟，确保测试的稳定性和可靠性。

🔍

元素定位策略

优先使用稳定的元素选择器，如 ID、数据属性等，避免使用易变的 XPath 或 CSS 路径。

📊

性能监控集成

将性能监控集成到持续集成流程中，及时发现和解决性能问题。

🔄

错误处理机制

实现完善的错误处理机制，记录详细的错误信息，便于调试和问题定位。

🔒

安全考虑

避免在测试脚本中硬编码敏感信息，使用环境变量或配置文件管理凭据。
